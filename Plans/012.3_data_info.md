## Goal

Add a "Scan Data" button on the Instruments page that triggers the worker to scan its historical data directory, report per-symbol date ranges back to the API, and display `data_from` / `data_to` columns in the Instruments table. This enables future phases (backtest date picker) to constrain the selectable date range to actual available data.

## Design Notes

- **Worker approach (Approach A):** Reuses existing job polling infrastructure. New task type `data-info` alongside `simple`, `complete`, `montecarlo`.
- **Symbol matching:** Worker data files use `@ES_1M_edit.txt` pattern. Need to map `@ES` ‚Üí `ES` (strip `@` prefix) to match Instrument symbols.
- **File scanning:** `DataPreprocessor` already has file discovery logic ‚Äî reuse or adapt it. Read first/last row dates from each matched file.
- **Job queue:** The `data-info` job goes through the same `backtest_jobs` table. May need a new `mode` value or a separate lightweight mechanism.
- **Future:** Phase 12.4 or backtest update will use `data_from`/`data_to` to set max date range in the backtest date picker.
